[**Math System**](https://mathsystem.dev/) 은 LaTeX·수식과 수학 글쓰기를 다루는 곳입니다. LaTeX 문법을 정리한 글과, 수학 문서 작업의 마찰을 줄이는 도구를 함께 만듭니다.

## 누가 만드나

수학·수식을 다루는 도구(LaTeXFlow)를 만드는 **독립 개발자**가 직접 운영합니다. 회사 조직이나 콘텐츠 외주가 아니라, 도구를 만들며 매일 부딪히는 문제를 그대로 글로 옮기는 1인 프로젝트입니다. 그래서 이 블로그의 글은 대부분 "직접 겪고 해결한 것" 에서 출발합니다 — Google Docs 에 적은 수식이 발표 자료에서 깨졌던 경험, Word 논문 초안의 수식 수십 개를 손보던 마감 직전의 밤, 챗봇 답변을 붙였더니 대괄호만 남던 순간처럼요.

문의는 [sung2417@gmail.com](mailto:sung2417@gmail.com) 또는 [GitHub](https://github.com/Seong6496/google-docs-latex-addon/issues) 으로 받습니다.

## 무엇을 만드나

- **블로그** — LaTeX 문법과 수식 표기 관례, 그리고 Google Docs·Word 에서 수학 글쓰기를 할 때의 노하우. 두 갈래로 씁니다. 하나는 그리스 문자·수식 오류처럼 **찾아 쓰는 레퍼런스**, 다른 하나는 실제 도구를 쓰며 겪은 **워크플로 기록**입니다.
- **[LaTeXFlow](/latexflow/web/)** — `.docx` 안의 LaTeX 수식(`$…$` / `$$…$$` / `\(…\)` / `\[…\]` / `[…]`)을 PNG 이미지로 변환하는 브라우저 도구. 문서가 브라우저를 벗어나지 않고, 원본 LaTeX 은 이미지 alt 텍스트에 보존됩니다. Google Docs 애드온도 있습니다.
- **Mathfolis** (개발 중) — 같은 문제가 없는 수학 문제집을 만드는 도구.

도구는 모두 *수학을 다루는 사람* 의 일상 작업 마찰을 줄이는 게 목적입니다.

## 글을 쓰는 원칙

- **직접 씁니다.** 외부 피드를 긁어 오거나 자동 생성한 글을 채우지 않습니다. 모든 글은 직접 작성하고, 예제는 실제로 컴파일·변환해 확인합니다.
- **경험을 담습니다.** 문법 요약에 그치지 않고 "언제 어떤 걸 쓰나 / 어디서 자주 막히나 / 왜 그런가" 를 함께 적으려 합니다.
- **업데이트합니다.** 도구가 바뀌거나 관례가 달라지면 글을 고칩니다. LaTeX 엔진·패키지 버전에 따라 결과가 다를 수 있다는 점은 [Disclaimer](/blog/disclaimer/) 에 정리해 두었습니다.
